# 🎯 Guía de Live Coding -- Templates y Rendering en Vue 3

> ⚠️ Esta guía contiene **pistas estratégicas**, no la solución
> completa. El objetivo es reforzar los conceptos vistos en la lección
> anterior.

---

# 🧩 ¿En qué consistirá la Demo?

Vamos a construir un componente que incluya:

- Renderización de listas con `v-for`
- Renderización condicional con `v-if` y `v-show`
- Formularios con `v-model`
- Eventos con modificadores
- Uso de abreviaciones `:` y `@`

La idea es aplicar la lógica declarativa y el sistema de rendering
reactivo de Vue.

---

# 🪜 Paso 1 -- Definir el Estado

📌 Pistas:

- El estado debe ser **reactivo**.
- Necesitas una lista inicial de usuarios.
- ¿Usarás `data()` (Options API) o `ref()` (Composition API)?
- ¿Cómo defines un array de objetos usuario?

Ejemplo conceptual (estructura, no solución final):

```js
data() {
  return {
    usuarios: []
  }
}
```

🔎 Documentación oficial:
https://vuejs.org/guide/essentials/reactivity-fundamentals.html

---

# 🔁 Paso 2 -- Mostrar Usuarios con v-for

📌 Pistas:

- `v-for` requiere una fuente iterable.
- No olvides usar `:key`.
- ¿Qué propiedad del usuario puede servir como identificador único?

Estructura mental:

```html
<li v-for="item in lista" :key="item.id">{{ item.nombre }}</li>
```

Pregunta guía:

¿Por qué Vue recomienda usar `key`?

🔎 Documentación oficial: https://vuejs.org/guide/essentials/list.html

---

# ❓ Paso 3 -- Mostrar Mensaje si la Lista Está Vacía

📌 Pistas:

- Evalúa la longitud del array.
- ¿Conviene usar `v-if` o `v-show`?
- ¿Quieres eliminar el nodo del DOM o solo ocultarlo?

Ejemplo conceptual:

```html
<p v-if="usuarios.length === 0">No hay usuarios registrados</p>
```

Reflexión:

¿Cuál es más eficiente si la condición cambia muchas veces?

🔎 Documentación oficial:
https://vuejs.org/guide/essentials/conditional.html

---

# 📝 Paso 4 -- Crear Formulario con v-model

📌 Pistas:

- `v-model` crea sincronización bidireccional.
- Necesitas una variable reactiva vinculada al input.
- ¿Qué sucede en el estado cuando el usuario escribe?

Ejemplo conceptual:

```html
<input v-model="nuevoUsuario" />
```

Pregunta:

¿Necesitas actualizar el DOM manualmente?

🔎 Documentación oficial: https://vuejs.org/guide/essentials/forms.html

---

# 🚀 Paso 5 -- Evento @submit.prevent

📌 Pistas:

- Los formularios recargan la página por defecto.
- ¿Qué modificador evita ese comportamiento?
- ¿Dónde se coloca el modificador?

Sintaxis mental:

```html
<form @submit.prevent="agregarUsuario"></form>
```

🔎 Documentación oficial:
https://vuejs.org/guide/essentials/event-handling.html

---

# ✋ Paso 6 -- Modificadores .stop y .once

📌 Pistas:

- `.stop` evita propagación del evento.
- `.once` ejecuta el evento una sola vez.

Ejemplo conceptual:

```html
<button @click.once="mostrarMensaje">
  <li @click.stop="seleccionarUsuario"></li>
</button>
```

Reflexión:

¿Qué ocurre si haces clic varias veces?

🔎 Documentación oficial:
https://vuejs.org/guide/essentials/event-handling.html#event-modifiers

---

# 👁 Paso 7 -- Mostrar Elemento Extra con v-show

📌 Pistas:

- `v-show` no elimina el nodo.
- Solo cambia `display: none`.
- Ideal para toggles frecuentes.

Ejemplo conceptual:

```html
<p v-show="mostrarInfo">Información adicional</p>
```

Pregunta clave:

¿Este elemento cambiará muchas veces durante la demo?

🔎 Documentación oficial:
https://vuejs.org/guide/essentials/conditional.html#v-show

---

# ✂ Paso 8 -- Abreviaciones :, @

Recordatorio:

- `v-bind:` → `:`
- `v-on:` → `@`

Ejemplo equivalente:

```html
v-bind:href="url" :href="url" v-on:click="accion" @click="accion"
```

📌 Reflexión didáctica:

En entorno educativo puede usarse la forma extendida.\
En producción se usa la abreviada por claridad y concisión.

🔎 Documentación oficial:
https://vuejs.org/guide/essentials/template-syntax.html
