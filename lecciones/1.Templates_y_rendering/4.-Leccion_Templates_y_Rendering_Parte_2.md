# 🧠 Lección: Templates y Rendering (Parte 2)

## 1️⃣ ¿Qué es v-for en Composition API?

`v-for` es una directiva que renderiza listas dinámicamente a partir de datos reactivos.
📖 **Docs:** [Vue.js List Rendering](https://vuejs.org/guide/essentials/list.html)

### ✅ Ejemplo básico con `<script setup>`

```vue
<script setup>
import { ref } from 'vue';

// ref crea una variable reactiva
const productos = ref([
  { id: 1, nombre: 'Laptop' },
  { id: 2, nombre: 'Mouse' },
  { id: 3, nombre: 'Teclado' },
]);
</script>

<template>
  <ul>
    <!-- 
      producto = cada elemento del array
      :key es obligatorio para el Virtual DOM
    -->
    <li v-for="producto in productos" :key="producto.id">
      {{ producto.nombre }}
    </li>
  </ul>
</template>
```

🔎 **¿Qué está pasando?**

1. `ref()` hace reactivo el array.
2. Vue detecta cambios automáticamente.
3. Se renderiza un `<li>` por cada producto.

---

## 2️⃣ v-for con índice

```vue
<li v-for="(producto, index) in productos" :key="producto.id">
  
  <!-- index representa la posición (0, 1, 2...) -->
  {{ index + 1 }} - {{ producto.nombre }}
</li>
```

> **Tip:** Útil para numeraciones o aplicar estilos alternos según la posición.

---

## 3️⃣ v-for con objetos

```vue
<script setup>
import { reactive } from 'vue';

// reactive para objetos complejos
const usuario = reactive({
  nombre: 'Maxi',
  rol: 'Admin',
  activo: true,
});
</script>

<template>
  <div v-for="(valor, clave, indice) in usuario" :key="clave">
    <!-- clave → propiedad | valor → contenido -->
    {{ indice }} - {{ clave }}: {{ valor }}
  </div>
</template>
```

## 4️⃣ v-if (Renderización Condicional)

`v-if` elimina o crea el nodo en el DOM según la condición.

```vue
<script setup>
import { ref } from 'vue';

const activo = ref(true);
</script>

<template>
  <!-- Solo se renderiza si activo es true -->
  <p v-if="activo">Usuario activo</p>
</template>
```

### ⚠ v-if + v-for

**Regla de oro:** No los uses en el mismo elemento. Es mejor usar una `computed` para filtrar los datos antes de enviarlos al template.

```vue
<script setup>
import { ref, computed } from 'vue';

const items = ref([
  { id: 1, nombre: 'A', activo: true },
  { id: 2, nombre: 'B', activo: false },
]);

// Filtramos antes de renderizar para mejor rendimiento
const activos = computed(() => items.value.filter((i) => i.activo));
</script>

<template>
  <li v-for="item in activos" :key="item.id">
    {{ item.nombre }}
  </li>
</template>
```

✔ **Más limpio** | ✔ **Mejor rendimiento**

## 5️⃣ Componentes dinámicos con `<component>`

Permite cambiar qué componente se renderiza en un mismo lugar.

```vue
<script setup>
import { ref } from 'vue';
import TabInicio from './TabInicio.vue';
import TabPerfil from './TabPerfil.vue';

const componenteActual = ref('TabInicio');

function cambiarTab(nombre) {
  componenteActual.value = nombre;
}
</script>

<template>
  <button @click="cambiarTab('TabInicio')">Inicio</button>
  <button @click="cambiarTab('TabPerfil')">Perfil</button>

  <!-- Renderiza dinámicamente según el valor de :is -->
  <component :is="componenteActual"></component>
</template>
```

## 6️⃣ keep-alive (Mantener estado)

```vue
<keep-alive>
  <component :is="componenteActual"></component>
</keep-alive>
```

🔎 Evita que Vue destruya el componente al cambiarlo, preservando el estado (ej. texto en inputs).
Ideal para: **Formularios, Tabs y Dashboards.**

## 7️⃣ Directivas Personalizadas (Composition API)

En Vue 3 se registran así:

```javascript
// main.js
app.directive('focus', {
  mounted(el) {
    el.focus();
  },
});
```

**Uso en el componente:**

```vue
<input v-focus />
```

## 8️⃣ Slots (Inyección de Contenido)

Permiten que el padre inyecte contenido HTML o componentes dentro del hijo.

### Hijo (Card.vue)

```vue
<template>
  <div class="card">
    <slot></slot>
    <!-- Punto de inyección -->
  </div>
</template>
```

### Padre

```vue
<Card>
  <h2>Título dinámico</h2>
</Card>
```

### Slot nombrado

**Hijo:** `<slot name="header"></slot>`
**Padre:** `<template #header><h1>Encabezado</h1></template>`

## 🔟 v-html (HTML dinámico)

⚠ **Atención:** Solo usar con contenido confiable para evitar ataques XSS.

```vue
<template>
  <div v-html="contenidoHTML"></div>
</template>
```

## 🎓 Resumen Conceptual

| Concepto        | Qué hace                                      |
| :-------------- | :-------------------------------------------- |
| `ref()`         | Hace reactiva una variable primitiva.         |
| `reactive()`    | Hace reactivo un objeto.                      |
| `computed()`    | Calcula valores derivados de forma eficiente. |
| `v-for`         | Renderiza listas.                             |
| `v-if`          | Renderiza condicionalmente.                   |
| `component :is` | Renderiza componentes dinámicos.              |
| `keep-alive`    | Preserva el estado de componentes inactivos.  |
| `slot`          | Permite composición flexible de contenido.    |

---
