# 🧑‍💻 Lección: Manejo de Eventos en Vue 3

## 🎯 Objetivo de Aprendizaje

Al finalizar la clase, el estudiante será capaz de:

-   Comprender qué es un **evento del DOM**
-   Manejar eventos usando **`v-on` y `@`**
-   Definir **métodos manejadores**
-   Usar **parámetros y `$event`**
-   Aplicar **modificadores de eventos**
-   Manejar **eventos de teclado**
-   Implementar **validación de formularios**
-   Comprender la relación entre **eventos y `v-model`**

------------------------------------------------------------------------

# 1️⃣ Introducción

Las aplicaciones web modernas reaccionan constantemente a interacciones
del usuario.

Ejemplos:

-   Hacer clic en un botón
-   Escribir en un input
-   Enviar un formulario
-   Presionar una tecla

Vue permite manejar estas interacciones mediante **directivas
declarativas**.

La principal es:

    v-on

Forma corta:

    @click

------------------------------------------------------------------------

# 2️⃣ ¿Qué es un evento del DOM?

Un **evento DOM** es una señal generada cuando el usuario interactúa con
una página web.

Ejemplos comunes:

  Evento      Descripción
  ----------- ---------------------------
  click       Usuario hace clic
  input       Usuario escribe
  submit      Envío de formulario
  keydown     Presiona una tecla
  mouseover   Mouse pasa sobre elemento

------------------------------------------------------------------------

# 3️⃣ Primer ejemplo: Evento click

## App.vue

``` vue
<script setup>

import { ref } from 'vue'

// variable reactiva
const contador = ref(0)

// función que se ejecuta al hacer click
function incrementar() {
  contador.value++
}

</script>

<template>

<h2>Contador</h2>

<p>{{ contador }}</p>

<button v-on:click="incrementar">
Incrementar
</button>

</template>
```

------------------------------------------------------------------------

# 4️⃣ Forma corta de eventos

Forma larga:

``` vue
v-on:click="incrementar"
```

Forma corta:

``` vue
@click="incrementar"
```

Ejemplo:

``` vue
<button @click="incrementar">
Incrementar
</button>
```

------------------------------------------------------------------------

# 8️⃣ Acceso al objeto evento

Vue permite acceder al objeto del evento mediante `$event`.

``` vue
<script setup>

function mostrarEvento(event){

console.log(event)
console.log(event.target)

}

</script>

<template>

<button @click="mostrarEvento">
Ver evento
</button>

</template>
```

# 1️⃣2️⃣ Modificadores de eventos

## prevent

Evita comportamiento por defecto.

``` vue
<form @submit.prevent="enviar">
```

## stop

Evita propagación del evento.

``` vue
<button @click.stop="accion">
```

## once

Ejecuta solo una vez.

``` vue
<button @click.once="saludar">
Solo una vez
</button>
```

## self

``` vue
<div @click.self="cerrar">
```

# 1️⃣4️⃣ Validación de formulario

``` vue
<script setup>

import { ref } from 'vue'

const nombre = ref("")
const error = ref("")

function validar(){

if(nombre.value.length < 3){
error.value = "El nombre debe tener al menos 3 caracteres"
}else{
error.value = ""
}

}

</script>

<template>

<h2>Formulario</h2>

<input
v-model="nombre"
@input="validar"
placeholder="Nombre"
>

<p v-if="error">
{{ error }}
</p>

</template>
```

------------------------------------------------------------------------

# 1️⃣5️⃣ Eventos y v-model

Este código:

``` vue
<input v-model="mensaje">
```

Equivale a:

``` vue
<input
:value="mensaje"
@input="mensaje = $event.target.value"
>
```

# 1️⃣6️⃣ Eventos personalizados

## BotonContador.vue

``` vue
<script setup>

const emit = defineEmits(["incrementar"])

function enviarEvento(){
emit("incrementar")
}

</script>

<template>

<button @click="enviarEvento">
Incrementar
</button>

</template>
```

## App.vue

``` vue
<script setup>

import { ref } from 'vue'
import BotonContador from './components/BotonContador.vue'

const contador = ref(0)

function aumentar(){
contador.value++
}

</script>

<template>

<h2>{{ contador }}</h2>

<BotonContador @incrementar="aumentar"/>

</template>
```

# 🧠 Resumen

Vue maneja eventos usando:

    v-on:evento

o

    @evento

Permite:

-   manejar clics
-   detectar teclas
-   validar formularios
-   comunicar componentes
-   usar modificadores

Esto permite crear **interfaces reactivas, limpias y mantenibles**.
